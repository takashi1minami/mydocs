# 高専門性データサイエンス案件のマッチング先と活用方針

## 1. 基本方針

業務時間外に、ゆるく、ただし高い専門性を活かしてデータサイエンス案件を受ける場合、マッチング先は大きく6系統に分けて考えるとよい。

狙うべき案件は、単なるデータ集計やExcel作業ではなく、以下のようなもの。

- データ品質診断
- 分析可能性診断
- Jupyter Notebook / Python分析コードレビュー
- 自由記述アンケートのLLM構造化
- 研究データの前処理・可視化
- 小規模ML PoCの評価設計レビュー
- 社会課題データの可視化
- 論文データ・研究データの構造化

重要なのは、**案件に合わせて何でも受けるのではなく、自分の商品メニューを固定しておくこと** である。

---

## 2. マッチング先の全体像

| 優先 | 場所 | 向いている案件 |
|---:|---|---|
| 1 | ココナラ / Lancersパッケージ | 固定メニュー型の分析診断・Notebookレビュー |
| 2 | クラウドワークス / Lancers案件応募 | CSV/Excel、可視化、アンケート、研究データ解析 |
| 3 | YOUTRUST / Wantedly / SOKUDAN | 高専門性の副業・業務委託 |
| 4 | GRANT / サービスグラント | NPO・地域団体・社会課題型、実績作り |
| 5 | note / Zenn / GitHub経由 | 専門性を見せて直依頼を受ける |
| 6 | Upwork / Toptal / Freelancer | 英語対応できるなら高単価・専門案件 |

最初は、以下の順で進めるのが現実的。

```text
ココナラ / Lancersで固定メニューを出す
  → GitHub / note / Zennで専門性を見せる
  → GRANT等で社会課題型の実績を作る
  → SOKUDAN / YOUTRUSTで高単価の副業案件を見る
```

## 3. 固定メニューを出す場所

## ココナラ / Lancersパッケージ

### 向いている理由

業務時間外で対応するなら、案件ごとに毎回ゼロから提案するより、最初から固定メニューとして出しておく方がよい。

出品型であれば、以下を事前に決められる。

- 対応範囲
- 価格
- 納期
- 修正回数
- 対象外事項
- 納品物
- 入力データの条件

これにより、低単価作業や要件膨張に巻き込まれにくくなる。

### 出品メニュー例

| メニュー名 | 価格帯 |
|---|---:|
| データ品質・分析可能性診断 | 49,800〜148,000円 |
| Jupyter Notebook / Python分析コードレビュー | 29,800〜79,800円 |
| 研究データの前処理・可視化Notebook作成 | 49,800〜148,000円 |
| アンケート自由記述のLLM分類・構造化 | 49,800〜148,000円 |
| 論文Abstractからの情報抽出テンプレート作成 | 49,800〜148,000円 |

### 出品時のポイント

低単価に見える表現は避ける。

避けたい表現：

- データ集計します
- Excel分析します
- Pythonでグラフ作ります
- アンケートまとめます

使いたい表現：

- データ品質診断
- 分析可能性診断
- Notebookレビュー
- LLM自由記述構造化
- 研究データ可視化
- 小規模ML PoC評価設計レビュー

同じ作業でも、「作業代行」ではなく **診断・構造化・評価設計** として売ると、高専門性を伝えやすい。

## 4. 案件応募型の場所

## クラウドワークス / Lancers

### 向いている用途

- 初期実績作り
- 小規模分析案件の探索
- CSV/Excel自動化案件の探索
- アンケート分析案件の探索
- データ可視化案件の探索
- 研究データ解析案件の探索

### 受けてよい案件

| 案件 | 理由 |
|---|---|
| データ品質診断 | 高専門性を出せる |
| Python分析コードレビュー | 非同期で対応しやすい |
| 研究データ可視化 | 知的資産化しやすい |
| アンケート分析 | 成果物が明確 |
| 自由記述分類 | LLM活用の専門性を出せる |
| 公開データ可視化 | 社会課題型に向く |
| 小規模な時系列分析 | 高単価化しやすい |
| PoC評価設計レビュー | 単なる作業代行になりにくい |

### 避けた方がよい案件

| 案件 | 理由 |
|---|---|
| 1万円以下で大量作業 | 投資対効果が悪い |
| 修正無制限 | 時間が読めない |
| 要件が曖昧 | 工数が膨らむ |
| 個人情報が未整理 | リスクが高い |
| 医療診断や患者データ分析 | 規制・責任が重い |
| 投資助言・自動売買 | 金融トラブルリスク |
| 大規模スクレイピング | 法務・規約リスク |
| 企業AI導入支援 | 本業競合リスク |
| 生成AI/RAG基盤構築 | 本業と近い |

### 提案時の注意

案件応募型では、プロフィールよりも提案文でスコープを切る力が重要である。

提案時には、以下を必ず書く。

- 対応範囲
- 納品物
- 前提条件
- 対象外
- 修正回数
- 納期
- 個人情報の扱い
- 成果保証ではないこと

## 5. 高専門性の副業・業務委託を探す場所

## SOKUDAN / YOUTRUST / Wantedly / レバテック / Indeed

### 向いている案件

クラウドワークス型より少し業務委託寄りになるが、高単価を狙いやすい。

向いている案件は以下。

- データ分析レビュー
- データ品質診断
- ML PoCレビュー
- KPI設計レビュー
- 時系列・異常検知PoC
- Python分析支援
- LLMを使った文書分類
- 研究・実験データ解析
- 小規模SaaSの分析設計

### プラットフォーム別の特徴

| プラットフォーム | 特徴 | 向き不向き |
|---|---|---|
| SOKUDAN | リモート・副業・業務委託案件が多い | 月10〜20時間程度の案件探しに向く |
| YOUTRUST | 人脈・副業オファー型 | 専門性が伝わるプロフィールが重要 |
| Wantedly | スタートアップ・NPO・ミッション系も探せる | 社会課題型と相性あり |
| レバテックフリーランス | 高単価案件が多い | 稼働が重い案件も多いため注意 |
| Indeed | 業務委託検索に使える | 条件比較用に使いやすい |
| 週末ワーカー | 副業特化 | 小さめの副業案件を探しやすい |

### 条件として明記すべきこと

高専門性の業務委託では、最初から以下を明記する。

- 月10〜20時間まで
- 非同期中心
- 土日・平日夜対応
- 診断・レビュー・PoC評価中心
- 本番運用なし
- 長期常駐なし
- 企業AI導入支援ではない
- 医療診断・投資助言は対象外

## 6. 社会課題型のマッチング先

## GRANT / サービスグラント / プロボノ系

### 向いている対象

- NPO
- 地域団体
- 一般社団法人
- 教育支援団体
- 子育て支援団体
- 高齢者支援団体
- 防災・環境・福祉系団体
- 地域活動団体

### 向いているテーマ

- 活動実績の可視化
- 助成金報告用グラフ作成
- アンケート自由記述分類
- 支援者/参加者データの集計
- 公開統計を使った地域課題可視化
- NPO向けデータ品質診断
- 活動インパクトの可視化
- 事業報告書向けデータ整理

### 使い方

プロボノ系は、無償または低報酬の案件が多い。

そのため、直接収益化よりも以下を目的に使うのがよい。

- 社会課題型案件の実績作り
- NPO向けデータ可視化のポートフォリオ作り
- 匿名化したサンプルレポート作り
- 将来の有料メニューの検証
- 社会課題型クライアントのニーズ理解

### 収益化へのつなげ方

```text
プロボノで1〜2件実績を作る
  → 成果物を匿名化してポートフォリオ化
  → NPO向け活動データ可視化パックとして出品
  → ココナラ / Lancers / noteで販売導線を作る
```

## 7. 研究者・学生・ヘルスケア/材料系に寄せる場所

南さんのヘルスケア・マテリアルインフォ関心を活かすなら、一般的なクラウドソーシングだけではなく、研究支援・論文データ整理の導線を作るとよい。

### 使える場所

| 場所 | 使い方 |
|---|---|
| note | 研究データ可視化、材料論文構造化の記事を書く |
| Zenn | Python Notebook、論文抽出CLI、LLM構造化の技術記事を書く |
| GitHub | BioMaterial Paper Minerのサンプル公開 |
| X / LinkedIn | 研究支援・材料データ構造化の発信 |
| 大学・研究者コミュニティ | 公開ツール経由で接点を作る |
| ResearchGate | 研究支援の実績・関心テーマを見せる |
| Kaggle | Notebookやデータ処理力を見せる |

### 出すべきコンテンツ例

- 論文Abstractから材料名・用途・評価指標を抽出するPythonテンプレート
- バイオマテリアル論文をLLMで構造化するスキーマ
- 研究データの前処理・可視化Notebookサンプル
- 小標本実験データを扱うときの注意点
- NPOの自由記述アンケートをLLMで分類する実例
- 公開統計を使った地域課題可視化サンプル
- BioMaterial Paper MinerのMVP

この領域は、マッチングサイトで案件を待つより、**GitHubと記事で専門性を見せて直依頼を作る** 方が向いている。

## 8. 海外プラットフォーム

## Upwork / Toptal / Freelancer

英語対応が可能であれば、海外プラットフォームも選択肢になる。

### 向いているテーマ

汎用データ分析では競争が激しいため、専門領域を絞るのがよい。

例：

- LLM-based survey coding
- research data cleaning and visualization
- biomaterials paper mining
- scientific literature data extraction
- ML PoC review
- time series anomaly detection
- anomaly detection for sensor data
- data quality assessment
- Python notebook review

### プラットフォーム別の特徴

| プラットフォーム | 特徴 |
|---|---|
| Upwork | 案件数が多い。競争も強い |
| Toptal | 高単価だが審査が厳しい |
| Freelancer | 幅広い案件があるが低単価も多い |

### 注意点

海外では、最初の実績作りが大変である。

そのため、最初から汎用データ分析で戦うのではなく、以下のような専門性を出す。

- LLM × survey classification
- biomaterials × literature mining
- research data visualization
- data quality diagnostics
- ML evaluation review

## 10. 各場所に載せるプロフィール文

以下は、ココナラ、Lancers、クラウドワークス、YOUTRUST、Wantedly等で使い回せるプロフィール文である。

```text
Python / pandas / scikit-learn / LLMを用いた、小規模データ分析・データ品質診断・分析Notebookレビュー・自由記述データの構造化を行います。

得意領域は、単なる集計作業ではなく、データが分析や予測に使える状態かを確認し、欠損・外れ値・定義揺れ・データリーク・評価指標の妥当性を整理することです。

中小企業、個人事業主、NPO、研究者向けに、短期・非同期・成果物明確な形で対応します。

対応例：
・データ品質・分析可能性診断
・Jupyter Notebook / Python分析コードレビュー
・アンケート自由記述のLLM分類
・研究データの前処理・可視化
・公開統計データの可視化
・小規模ML PoCの評価設計レビュー

対応対象外：
・医療診断
・投資助言
・個人情報を含む未匿名化データ分析
・生成AI基盤構築
・企業向けAI導入コンサル
・長期常駐型案件
```

## 12. 提案文テンプレート

案件応募時には、以下のような提案文を使う。

```text
はじめまして。Python / pandas / scikit-learn / LLMを用いた、小規模データ分析・データ品質診断・分析Notebookレビューに対応できます。

今回のご依頼内容であれば、単なる集計作業ではなく、データが分析目的に対して使える状態かを確認し、欠損・外れ値・定義揺れ・分析リスクを整理したうえで、再利用しやすい形に整えることが可能です。

【対応内容】
・データ構造の確認
・欠損、重複、外れ値、表記ゆれの確認
・分析目的との整合性確認
・必要に応じた可視化
・簡易レポート作成
・再利用可能なNotebookまたは手順書の作成

【納品物】
・診断レポート
・集計/可視化ファイル
・必要に応じてPython Notebook
・改善提案リスト

【前提】
・個人情報は匿名化またはマスキングをお願いします
・医療診断、投資助言、成果保証に該当する内容は対応対象外です
・修正は1回までを想定しています
・データ受領後、内容に応じて納期を確定します

まずはサンプルデータ、列名、分析目的を共有いただければ、対応範囲を確認いたします。
```

## 13. 避けるべきマッチング・案件

高専門性を活かす場合でも、以下は避ける。

| 避けるもの | 理由 |
|---|---|
| 安すぎる大量作業 | 知的資産化しにくい |
| 修正無制限 | 稼働が読めない |
| 要件が曖昧なAI案件 | 本業競合や工数増加につながる |
| 個人情報未処理データ | リスクが高い |
| 医療診断・患者データ分析 | 医療規制・責任が重い |
| 投資助言・売買判断 | 金融規制・トラブルリスク |
| 本番システム開発 | 業務時間外には重い |
| 長期常駐型案件 | 本業に影響する |
| 成果報酬型 | 責任範囲が曖昧 |
| 大規模スクレイピング | 規約・法務リスク |

## 3〜4週目

- GitHubにサンプルNotebookを公開する
- noteまたはZennに記事を書く
  - データ品質診断の観点
  - 自由記述分類の例
  - 研究データ可視化の例
- クラウドワークス / Lancersで条件に合う案件だけ応募する
- 低単価すぎる案件は避ける

## 15. 最終結論

高専門性データサイエンス案件のマッチング先としては、以下の使い分けがよい。

| 目的 | 場所 |
|---|---|
| 小さく固定メニューで売る | ココナラ、Lancersパッケージ |
| 案件に応募して実績を作る | クラウドワークス、Lancers |
| 高単価・副業業務委託を見る | SOKUDAN、YOUTRUST、Wantedly、レバテック |
| 社会課題型の実績を作る | GRANT、サービスグラント |
| 専門性で直依頼を作る | GitHub、note、Zenn、LinkedIn、X |
| 海外高単価を狙う | Upwork、Toptal、Freelancer |

南さんの場合、最初に取るべき行動は以下である。

> ココナラ / Lancersで固定メニューを出し、GitHub / note / Zennで専門性を見せ、GRANT等で社会課題型の実績を作る。

最初に出すべき固定メニューは以下。

1. データ品質・分析可能性診断
2. Jupyter Notebook / Python分析コードレビュー
3. 自由記述アンケートのLLM構造化

この3つは、業務時間外でも対応しやすく、高専門性を伝えやすく、本業競合リスクも抑えやすい。

最終的には、スポット案件をこなしながら、以下の知的資産を増やすのがよい。

- 診断チェックリスト
- サンプルNotebook
- レポート雛形
- LLM分類プロンプト
- 研究データ構造化スキーマ
- BioMaterial Paper MinerのようなGitHub資産
