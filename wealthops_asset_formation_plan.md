# コードを使って自分の資産形成に役立てる方向性

## 1. 基本方針

「コードを使って自分の資産形成に役立てる」方向性は、非常に相性が良い。

外部向けの副業や対人商売を先に考えるのではなく、まずは自分専用の **投資・資産形成OS** を作るとよい。

目的は、AIに「何を買うべきか」を聞くことではない。コードを使って、以下の流れを自動化・定型化することである。

```text
情報収集
  → 整理
  → 可視化
  → ルール確認
  → 投資メモ作成
  → 振り返り
```

投資判断そのものは人間が行い、コードは **判断の質と継続性を高める補助輪** として使う。

---

## 2. なぜこの方向性が合うのか

自分の経験・関心と以下の点で相性が良い。

- AI・データサイエンスが分かる
- クラウドインフラが分かる
- アーキテクチャ設計ができる
- Coding Agentで開発効率を上げられる
- 企業分析・フリーキャッシュフロー・技術戦略への関心がある
- 本業と競合しにくい
- 対人稼働を最小化できる
- 自分の資産形成に直接役立つ

本業と競合しやすい企業向けAI導入支援やクラウド支援ではなく、自分の家計・資産配分・投資メモ・公開情報ベースの企業分析に閉じることで、安全性も高い。

---

## 3. 作るべきものは「自動売買」ではなく「投資判断支援」

最初に避けるべきは、自動売買ボットである。

### 自動売買を避ける理由

- 売買ロジックを過信しやすい
- バックテストの過剰最適化になりやすい
- 相場環境が変わると崩れやすい
- 短期売買へ寄りやすい
- 税務・記録・損益管理が複雑になる
- 心理的負荷が高くなる

目指すべきは、以下である。

> 長期投資のための情報整理・資産配分・企業分析・振り返りツール

つまり、コードで直接儲けるのではなく、コードで **投資行動を規律化する**。

---

## 4. 作るべきプロダクト像

## Personal Wealth Intelligence System

自分の資産、入金計画、NISA枠、投資候補、決算情報、投資メモを統合管理する個人用ツール。

### 主な領域

| 領域 | 目的 |
|---|---|
| 資産管理 | 現金、NISA、特定口座、iDeCo、個別株、投信を見える化する |
| 入金管理 | 毎月いくら投資に回せるかを計算する |
| ポートフォリオ管理 | 目標配分からのズレを確認する |
| 企業分析 | 株価、決算、FCF、成長率、バリュエーション、技術戦略を整理する |
| 投資日誌 | 買った理由、売った理由、失敗理由を記録する |

これは「投資判断AI」ではなく、**投資判断のためのデータ基盤** である。

## 5. 最初に作るべき4つのモジュール

## Module 1：資産ダッシュボード

まずは自分の資産全体を見える化する。

### 入力データ

- 現金
- 銀行口座
- 証券口座
- NISA残高
- 投資信託
- ETF
- 個別株
- 外貨
- 副業収入
- 毎月の投資可能額

### 出力

- 総資産推移
- リスク資産比率
- 現金比率
- NISA利用状況
- 月次入金額
- 目標資産との差分

資産形成では「何を買うか」よりも、**いくら入金し続けられるか、リスクを取りすぎていないか** が重要である。

---

## Module 2：NISA枠・入金計画シミュレーター

NISA枠の利用状況と、将来の入金計画を管理する。

### 管理すること

- 毎月いくら入金すれば年間枠を使えるか
- 副業収入を何％投資に回すか
- 何年で生涯非課税枠を使い切るか
- 生活防衛資金を残した場合の投資可能額
- 暴落時に追加投資できる余力

### シミュレーション例

```text
毎月投資額：10万円
年間投資額：120万円
NISA 1,800万円枠を使い切る年数：15年

毎月投資額：20万円
年間投資額：240万円
NISA 1,800万円枠を使い切る年数：7.5年

毎月投資額：30万円
年間投資額：360万円
NISA 1,800万円枠を使い切る年数：5年
```

副業収入が発生する場合は、**副業収入の70〜100%をNISA投入候補にする** というルールを作れる。

---

## Module 3：ポートフォリオ・リバランス支援

目標配分と現在配分を比較し、リスクの偏りを確認する。

### 目標配分の例

| 資産 | 目標比率 |
|---|---:|
| 全世界株式 / 米国株式インデックス | 70% |
| 個別株・テーマETF | 15% |
| 現金・安全資産 | 15% |

### 出力

- リスク資産が多すぎないか
- 現金が少なすぎないか
- 個別株比率が上がりすぎていないか
- 次の入金で何を買うと目標に近づくか
- 売却せずに入金だけでリバランスできるか

ただし、ツールは売買指示を出さない。出すのは、あくまで以下に留める。

- 目標配分との差分
- 次回入金で調整するなら候補は何か
- リスクが高まっているか

## Module 4：株価・経営指標・AI/クラウド企業分析

自分の専門性を最も活かせるモジュール。

このモジュールの中心は、株価予測ではなく、**株価・経営指標・投資仮説を同じ時間軸で見ること** である。

### 対象企業例

- Microsoft
- NVIDIA
- Amazon
- Google
- Meta
- Oracle
- IBM
- Salesforce
- ServiceNow
- Palantir
- Snowflake
- Datadog
- Cloudflare
- CrowdStrike
- Broadcom
- TSMC

### コードで行うこと

1. 決算資料・10-K・10-Q・有価証券報告書を取得する
2. 売上、営業利益、FCF、利益率、成長率を抽出する
3. 株価、時価総額、PER、PSR、EV/EBITDAなどの推移を取得する
4. AI、クラウド、データセンター、セキュリティ関連の記述を抽出する
5. 過去数四半期の変化を比較する
6. 投資メモの雛形を生成する
7. 自分の投資仮説と照合する
8. 個別株とマーケットインデックスの推移パターンを比較する

### 取り込むべきデータ

| データ | 目的 |
|---|---|
| 株価推移 | 保有銘柄・ウォッチ銘柄の価格変化、ドローダウン確認 |
| 時価総額 | 企業規模と市場評価の変化を見る |
| 売上高 | 成長率の確認 |
| 営業利益 | 本業の収益力を見る |
| 営業利益率 | 事業の質の変化を見る |
| 純利益 / EPS | 最終利益と1株あたり利益の推移を見る |
| FCF | 現金創出力の確認 |
| FCFマージン | 高品質企業かどうかを見る |
| ROE / ROIC | 資本効率を見る |
| PER / PSR / EV/EBITDA | バリュエーションの変化を見る |
| ガイダンス | 会社側の見通し変化を見る |
| セグメント売上 | AI、クラウド、広告、半導体などの成長源を見る |
| マーケットインデックス | 個別株の変動が市場全体によるものか、企業固有要因かを見る |
| 金利・為替 | グロース株、米国株、外貨建て資産への環境変化を見る |

### 分析の問い

株価だけを見ると短期売買に寄りやすいため、必ず経営指標とセットで見る。

- 株価は上がったが、売上成長率やFCFも改善しているか
- 株価は下がったが、事業の質そのものは悪化しているか
- PERやPSRの上昇は利益成長によるものか、期待先行か
- AI関連投資は売上、利益率、FCFに効いているか
- 決算後の下落は一時的な期待外れか、投資仮説の崩れか
- 個別株の下落は市場全体の下落と連動しているか、企業固有の弱さか
- 同業他社やNASDAQ/S&P 500に対して相対的に強いか弱いか
- 買い増し候補に見える下落は、単なる地合い悪化か、事業悪化を伴う下落か

### 株価・マーケットパターン分析

買い時・売り時の判断を支援するために、個別株の株価だけでなく、その時点の市場環境も一緒に見る。

| 観点 | 見るもの |
|---|---|
| 個別株トレンド | 20日、60日、200日移動平均との位置 |
| ドローダウン | 直近高値からの下落率 |
| 相対パフォーマンス | S&P 500、NASDAQ、同業ETFとの差 |
| ボラティリティ | 直近の値動きが通常より大きいか |
| 出来高 | 決算・ニュース後に出来高を伴って動いたか |
| 市場局面 | インデックスが上昇基調、調整局面、急落局面のどれか |
| 金利・為替 | 米国金利、ドル円、VIXなどの環境変化 |

出力するのは売買シグナルではなく、以下のような判断材料に留める。

```text
MSFT:
  - 株価は60日移動平均を下回ったが、200日移動平均は上回っている
  - 直近高値から -12%
  - 同期間のNASDAQは -8%、S&P 500は -5%
  - 相対的には市場より弱い
  - 売上成長率とFCFマージンは前四半期比で悪化していない
  - 判断ステータス：バリュエーション確認 / 決算確認待ち
```

このモジュールでは、AIに売買判断をさせるのではなく、公開情報の整理と論点抽出をさせる。

### ローカルLLMの位置づけ

OpenAI APIなどの外部LLMは便利だが、個人資産データや投資日誌を扱うため、最初はOSSのローカルLLMを試す価値が高い。

```text
数値計算・資産集計
  → Python / pandas で確実に行う

文章要約・論点整理
  → OllamaなどのローカルLLMで補助する

売買判断
  → 人間が行う
```

ローカルLLMに任せる対象は、投資日誌の要約、決算資料の論点整理、ニュース分類、月次レポートの文章整形に留める。

## 8. AIの使いどころ

AIは投資判断そのものではなく、情報整理に使う。

| AI・MLに任せる | 人間がやる |
|---|---|
| 決算資料の要約 | 投資判断 |
| 企業の論点抽出 | 買う/売る/持つの判断 |
| リスク要因の整理 | リスク許容度の決定 |
| 過去メモとの比較 | ポートフォリオ配分の決定 |
| ニュース分類 | 投資仮説の採否 |
| 投資日誌の分類 | 自分の行動改善 |
| 異常値の検出 | どのリスクを受け入れるか |
| レポート下書き | 最終判断 |

### AIへの問いの例

```text
この決算資料から、売上成長、営業利益率、FCF、AI関連投資、
顧客需要、リスク要因に関する記述を抽出してください。

前回決算と比較して、事業の質が改善した点・悪化した点・判断保留の点を分けてください。

この企業について、私が過去に書いた投資仮説と矛盾する記述を抽出してください。
```

AIに聞くべきなのは「この株を買うべきか」ではなく、**判断材料の整理** である。

### 機械学習の使いどころ

このプロダクトの主役は機械学習ではなく、データエンジニアリングとルールベース分析である。ただし、以下の領域では機械学習を使う価値がある。

| 領域 | ML要素 | 優先度 |
|---|---|---:|
| 投資日誌の分類 | 売買理由、感情、失敗パターンの分類 | 中 |
| ニュース分類 | 決算、AI投資、規制、競争環境などへ分類 | 中 |
| 決算資料の文書分類 | リスク要因、成長要因、ガイダンス変更を抽出 | 中 |
| 異常検知 | 支出急増、ポートフォリオ偏り、入金力低下の検出 | 高 |
| 類似企業検索 | 似た事業モデル・財務特徴の企業を探す | 低〜中 |
| スコアリング | 財務健全性、成長性、集中リスクなどの指標化 | 中 |
| 株価予測 | 将来リターンや売買タイミングの予測 | 低 |

最初に入れるなら、予測ではなく、**分類・埋め込み検索・異常検知** がよい。

```text
投資日誌
  → 自分の判断理由を分類
  → 決算前の期待買い、下落時の感情売りなどを検出

ポートフォリオ
  → 銘柄、セクター、通貨、テーマの偏りを数値化
  → 過去比で異常な集中を検出

ニュース/決算資料
  → AI投資、クラウド成長、規制、競争激化などに分類
  → ウォッチリスト企業ごとに論点を蓄積
```

株価予測や売買タイミング予測は、過剰最適化しやすく、投資判断支援という目的から外れやすいため優先しない。

## E. 買い時・売り時レビュー支援

売買タイミングを予測するのではなく、判断前に見るべき情報を揃える。

### Buy Review

- なぜ今この企業を見るのか
- 株価下落は市場全体によるものか、企業固有要因か
- インデックスや同業他社に対して相対的に強いか弱いか
- 売上成長率、利益率、FCFは悪化していないか
- PER、PSR、FCF利回りは過去レンジのどこにあるか
- 目標ポートフォリオ上、買い増し余地があるか
- 生活防衛資金、NISA枠、入金計画に無理がないか
- 買わない理由は何か

### Sell Review

- 当初の投資仮説は何だったか
- 仮説が崩れた証拠はあるか
- 株価下落は市場全体と連動しているだけか、企業固有の悪化か
- 売上成長率、利益率、FCFが複数期で悪化しているか
- 個別株比率、テーマ比率、通貨比率が上がりすぎていないか
- 売却後の資金用途は明確か
- 単なる恐怖や焦りによる判断ではないか

### 判断ステータス

```text
- 調査継続
- 決算確認待ち
- バリュエーション確認
- 仮説強化
- 仮説悪化
- 市場連動の下落
- 企業固有の悪化
- 配分超過
- 感情判断リスクあり
```

## G. 投資日誌・行動バイアス検出

毎回の売買や判断に対して、以下を記録する。

- なぜ買ったか
- なぜ売ったか
- 何を期待したか
- 何が起きたら見直すか
- 感情的な判断ではなかったか

後からコードで以下を分析する。

- 決算前に買い増しして失敗していないか
- 下落時に感情的に売っていないか
- SNSで話題の銘柄に寄りすぎていないか
- 得意なAI/クラウド銘柄に集中しすぎていないか
- 買った理由と売った理由が一貫しているか

これは資産形成に非常に効く。

## 16. 結論

最も良い方向性は以下である。

> 自分用の資産形成支援ツールをコードで作る。  
> まずは資産管理・NISA進捗・ポートフォリオ配分・投資日誌を自動化する。  
> 次に株価・経営指標・AI/クラウド企業分析モジュールを追加する。  
> さらに、個別株の株価パターンとマーケットインデックスの推移を比較し、買い時・売り時レビューの材料を揃える。  
> 文章整理にはローカルLLMや機械学習を補助的に使う。  
> 最後に、個人情報と売買判断を除いたテンプレート部分だけをGitHub公開・販売する。

この方向性は、以下の条件を満たす。

- 本業競合リスクが低い
- 対人稼働が少ない
- 自分の資産形成に直接役立つ
- コード資産として積み上がる
- 株価、財務、投資仮説、判断履歴を同じ画面で確認できる
- 個別株の動きが市場連動か企業固有要因かを切り分けやすい
- 将来的にGitHub公開や有料テンプレート販売に展開できる

最初の一手は、`wealthops` として **CSVベースの資産ダッシュボード + NISA進捗 + 月次Markdownレポート生成** から始めるのがよい。

次の一手として、ウォッチリスト企業の **株価推移 + 主要経営指標 + マーケットインデックス + 投資仮説メモ** をSQLiteに保存し、株価だけでなく事業の質、市場評価、市場環境をセットで見られるようにする。
